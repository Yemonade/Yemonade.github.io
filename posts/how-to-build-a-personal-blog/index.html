How to Build a Personal Blog#

  1. Install Hugo

    # 输入下列命令检查是否安装成功
hugo version

  2. Create a Folder

    cd dir
hugo new site site_name -f yml

  3. Choose a Theme

    从Hugo官网进入，点击 Themes 下载喜欢的主题

    这里选择PaperMode主题

    然后Download: 在之前创建的site_name文件夹下输入下列命令即可

    git clone url地址 themes/PaperMod --depth=1

    在根目录下的config.yml增加

    theme: "PaperMod"

  4. 创建第一个post

    在根目录下输入下列命令

    hugo new posts/first_post.md

    可以发现根目录的content文件夹下多了posts/文件夹，里面存我们新建的 first_post.md 文件

    点击进入，将draft:true一行改为draft:false

  5. 在本地启动

    hugo server
# or hugo serve -e production

    然后输入网址http://localhost:1313就可以看到啦

  6. 部署到github

    在GitHub上创建一个仓库，名为Github用户名.github.io。注意命名规范

    在根目录执行：hugo

    此时，所有的静态页面都会生成到public目录。执行以下命令

    cd public
git init
git remote add origin git@github.com:Github用户名/Github用户名.github.io.git
git add -A
git commit -m "first commit"
git push -u origin master

    浏览器输入https://Github用户名.github.io/，就可以看到了

    注意：最后一步push失败可考虑

    法一：

    修改.git/config文件

    将 [remote “origin”] url = https://github.com/用户名/仓库名.git

    修改为 [remote “origin”] url = https://用户名:密码@github.com/用户名/仓库名.git

    若显示Connection was reset, errno 10054错误，解决方案

    git config --global --unset-all remote.origin.proxy

    法二：

    cd ~/.ssh
ssh-keygen -t rsa -C "邮箱地址"
cat id_rsa.pub

    添加公钥

    若发现push到master分支

    git checkout -b main  # 切换分支到main
git branch -D master  # 删除本地的master分支
git push origin --delete master 删除Github项目上的master分支
git push -u origin main -f # 强制进行代码推送（会失去readme文档）

  7. 剩下的一些事

    之后创建blog只需在之前说的那个/content/blog文档下创建新的文件即可，然后再输入下列命令即可

    hugo
cd public
git init
git remote add origin git@github.com:Github用户名/Github用户名.github.io.git
git add -A
git commit -m "xxx"
git push 
# 注意若无反应先hugo server一下，然后再打开Blog网站
